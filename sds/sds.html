  1. Introduction :

Purpose of the Document:

This design document describes all the data, architectural,interface and component level design for IVENTO. The design specification provides explicit information about requirements for the product and how the product will be put together.

  1. Document Convention:

        Web: html,css,bootstrap,Django

        Android : xml,java.

  1. Description of Design components:

3.1. Server Side: The server side component of IVENTO is implemented with  Django Framework.All information of product and event list will be stored through the database by using MYSQL.

3.2. Client Side: The Client side components are implemented with HTML5, Bootstrap and JavaScript to make the web pages responsive,and our web app also belongs to the client side .

Third party side:We use some reusable API for login,signup  ,AR view.

  1. High-level Component Design :

Description: HTML5 browser which is our client-side component depends on web UI and web UI also depends on back end which is our server side which is connected to 3rd party API. And these all mainly depend on the MYSQL database. The Android app also depends on MYSQL.
